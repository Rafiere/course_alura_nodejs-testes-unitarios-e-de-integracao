#Unitary tests

Unit tests are short, isolated tests. Typically, they are for parsing functions or methods.

It does not guarantee integration between application modules.

# Integration Tests

These are test routes and requests. They analyze the communication of the modules.

It does not analyze the entire application flow.

# E2E Tests

These are end-to-end tests. They are top notch. They are long and complete. It analyzes all modules and stacks such as frontend, database and etc.

It has a high cost and a complex testing structure.

# Test Phases

Requirements Analysis: What features and what types of tests will be implemented.
Test Plan: The QA team develops a test plan.
Test Case: The data and the way in which they will be carried out are detailed.
Test Environment: How and where the tests will be performed are chosen.
Implementation: Tests are implemented.
